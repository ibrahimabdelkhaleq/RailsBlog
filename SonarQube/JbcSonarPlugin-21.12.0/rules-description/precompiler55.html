This rule restricts the direct read of Standard Selection record as it will cause performance issue. Instead specific API can be used to read the SS record.

Noncompliant Code Example

SSRec = EB.SystemTables.StandardSelection.Read(RecId, Error)
CALL F.READ('F.STANDARD.SELECTION',id,rec,' ',error)

Compliant Solution

The following API can be used to get the details of Standard Selection instead of direct read.

CALL GET.STANDARD.SELECTION.DETS('SECTOR',SSRec)
EB.API.GetStandardSelectionDets('SECTOR', SsRec)
